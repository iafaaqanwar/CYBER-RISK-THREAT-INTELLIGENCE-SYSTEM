Threat Ingestion & Analysis Lifecycle

1
Step 1

Packet Ingestion & Feature Extraction

Network packets are captured in real-time (via pyshark) or parsed from uploaded .pcap files. Features including protocol, ports, packet length, and payload identifiers are extracted instantly.

2
Step 2

Machine Learning Anomaly Detection

Extracted features are scaled and predicted by a trained scikit-learn model. Safe traffic is logged directly, while anomalous traffic profiles are routed to the expert diagnostic engine.

3
Step 3

Prolog Expert Logic Diagnosis

The anomaly characteristics query the Prolog knowledge base (knowledge_base.pl). The expert system uses formal reasoning logic to pinpoint the attack vector and recommend precise mitigations.

4
Step 4

Encrypted Database Logging & Reports

Threat profiles are encrypted using AES-256-GCM and logged in the secure SQLite ledger. Forensics auditors can generate formal PDF reports for administrative review.
